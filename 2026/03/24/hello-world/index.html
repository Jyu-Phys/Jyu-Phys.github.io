Hello World

发表于2026-03-24|更新于2026-03-25
|浏览量:

Welcome to Hexo! This is your very first post. Check documentation for more info. If you get any problems when using Hexo, you can find the answer in troubleshooting or you can ask me on GitHub.

欢迎使用 Hexo！这是系统自动生成的第一篇文章。你可以把它当作你的 Hexo 常用核心指令备忘录。

Quick Start (快速开始)

Create a new post (新建文章)

用命令行新建一篇博客文章。文章的 .md 源文件会自动生成在 source/_posts 目录下。

(💡 简写形式：hexo n "你的文章标题")

More info: Writing

Clean cache (清理缓存)

中文提示： 每次修改了全局配置 _config.yml、修改了主题配置、换了新头像，或者发现网页样式“死活不更新”时，请务必第一时间执行这个指令！它的作用是彻底清除旧的缓存文件 (db.json) 和编译好的静态文件夹 (public)，打扫战场。

(💡 简写形式：hexo c)

Generate static files (生成静态文件)

将你写的 Markdown 文章和配置文件，编译打包成最终能被浏览器识别的 HTML 静态网页文件。

(💡 简写形式：hexo g)

More info: Generating

Run server (启动本地测试)

在本地电脑上启动一个测试服务器，用于在浏览器中实时预览你的博客效果（默认地址是 http://localhost:4000/）。预览结束后，在黑色命令行窗口按 Ctrl + C 即可关闭。

(💡 简写形式：hexo s)

More info: Server

Deploy to remote sites (一键部署上云)

将生成好的静态网页，一键推送到 GitHub Pages 等远程服务器上，发布给全网看。

(💡 简写形式：hexo d)

More info: Deployment

👑 终极万能连招 (Daily Workflow)

平时写完一篇科研笔记，或者改完一行配置代码后，闭着眼睛在黑窗口里按顺序敲这三个短命令就行了：

  1. hexo c (清空旧数据)
  2. hexo g (重新打包生成)
  3. hexo d (上传发布)
